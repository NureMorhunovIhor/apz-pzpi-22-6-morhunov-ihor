3 ОПИС ВИКОНАНОЇ РОБОТИ
Amazon Web Services (AWS) — це одна з найпопулярніших і найпотужніших хмарних платформ у світі, яка надає понад 200 різноманітних сервісів для роботи з обчисленнями, зберіганням даних, мережами, аналітикою, штучним інтелектом, безпекою та розробкою застосунків. Платформа була запущена компанією Amazon у 2006 році як інфраструктурне рішення для сторонніх розробників і бізнесу, а згодом виросла до глобальної багатофункціональної екосистеми для компаній будь-якого масштабу — від невеликих стартапів до гігантів на кшталт Netflix, Airbnb, NASA та McDonald’s. Основна ідея AWS — дати можливість користувачам орендувати хмарні ресурси замість купівлі власного обладнання та підтримки власних дата-центрів.
Ключовим елементом обчислювальної інфраструктури є Amazon EC2 (Elastic Compute Cloud) — сервіс для оренди віртуальних серверів (інстансів) із заданими характеристиками. Він дозволяє гнучко вибирати типи машин, операційні системи, розмір дисків та налаштування мережі. Для сценаріїв із подієвим запуском програмного коду використовується AWS Lambda — безсерверна платформа, яка автоматично виконує функції у відповідь на події без потреби запускати сервери. Контейнерні застосунки розгортаються за допомогою Amazon ECS (Elastic Container Service) або керованого Kubernetes-сервісу Amazon EKS.
Для зберігання даних в AWS передбачено кілька сервісів. Найпопулярніший — Amazon S3 (Simple Storage Service), який є об’єктним сховищем для файлів будь-якого формату, з підтримкою різних класів доступності, резервного копіювання та контролю доступу. Для блочного сховища даних використовується Amazon EBS (Elastic Block Store), яке забезпечує постійні дискові ресурси для EC2-інстансів. Якщо потрібен доступ до спільного файлового сховища, використовується Amazon EFS (Elastic File System), який дозволяє одночасно підключати одні й ті самі диски до різних віртуальних серверів.
У сфері баз даних AWS також має широкий набір рішень. Amazon RDS (Relational Database Service) — це керований сервіс для реляційних баз даних з підтримкою MySQL, PostgreSQL, Oracle, Microsoft SQL Server, MariaDB та власної оптимізованої системи Aurora. Для NoSQL-сховищ доступний Amazon DynamoDB — база даних для зберігання структур типу ключ-значення з низькою затримкою. Для аналітики великих обсягів даних використовується Amazon Redshift — високопродуктивна колоночна база даних, адаптована для складних SQL-запитів. Для роботи з кешем даних в оперативній пам’яті AWS пропонує ElastiCache з підтримкою Redis і Memcached, а для побудови графових структур і аналізу взаємозв’язків — Amazon Neptune.
Мережеву інфраструктуру в AWS реалізовано через Amazon VPC (Virtual Private Cloud), який дозволяє створювати ізольовані приватні мережі, налаштовувати маршрутизацію, VPN-з’єднання та міжмережеві екрани (Security Groups). Для балансування навантаження використовується Elastic Load Balancing (ELB), що підтримує розподіл HTTP(S), TCP і UDP трафіку. Для керування DNS-записами і маршрутизацією запитів — Amazon Route 53, який також виконує функції геолокаційного балансування. Для прискорення доставки контенту по всьому світу застосовується CDN-сервіс Amazon CloudFront.
Окрему нішу займають безсерверні інтеграції. Amazon API Gateway дозволяє швидко створювати та публікувати API для взаємодії між сервісами. AWS Step Functions використовується для оркестрації безсерверних та мікросервісних сценаріїв. Для побудови подієво-орієнтованих архітектур існує сервіс Amazon EventBridge, який дозволяє централізовано передавати події між сервісами. Для роботи з потоками даних у реальному часі передбачено Amazon Kinesis, а для обробки великих обсягів даних — Amazon EMR (Elastic MapReduce) і AWS Glue, який автоматизує ETL-процеси.
У сфері штучного інтелекту та машинного навчання AWS пропонує повнофункціональну платформу Amazon SageMaker, яка дозволяє створювати, тренувати, тестувати й розгортати ML-моделі з використанням Jupyter-ноутбуків, кластерів для тренування та автоматизованого розгортання. Окрім цього, AWS має набір готових сервісів для розпізнавання зображень і облич (Rekognition), синтезу мови (Polly), створення чат-ботів (Lex) та аналізу текстів (Comprehend).
Ще один важливий напрямок — автоматизація інфраструктури. AWS CloudFormation дозволяє описувати хмарну інфраструктуру як код за допомогою YAML-або JSON-шаблонів. Для реалізації повноцінних CI/CD-процесів AWS надає сервіси CodePipeline, CodeBuild, CodeDeploy та CodeCommit.
Безпека в AWS реалізована через сервіс IAM (Identity and Access Management), який дозволяє створювати користувачів, ролі, політики доступу й керувати правами на доступ до ресурсів. Для захисту вебзастосунків від атак типу DDoS і налаштування Web Application Firewall доступні AWS WAF і AWS Shield. Секрети й ключі шифрування зберігаються в AWS Secrets Manager та AWS KMS (Key Management Service).
Глобальна архітектура AWS побудована на основі 29 регіонів, 96 зон доступності та понад 400 точок присутності (Edge Locations) по всьому світу. Така структура дозволяє забезпечити низькі затримки, георезервування та високу відмовостійкість. У контексті технологій AWS активно використовує контейнери, серверлес-архітектуру, асинхронні черги, потокову обробку даних і кешування. Для моніторингу та аналізу інфраструктури використовується Amazon CloudWatch, який збирає логи, метрики й надсилає сповіщення.


4 ВИСНОВКИ

	Архітектура Amazon Web Services є високопродуктивною, масштабованою та надійною, що дозволяє обслуговувати мільйони користувачів і тисячі клієнтських систем по всьому світу. Використання сучасних технологій і підходів, таких як безсерверні обчислення, контейнери, машинне навчання, автоматизоване масштабування та інфраструктура як код, дозволяє AWS забезпечувати гнучке керування ресурсами, стабільну роботу сервісів і швидке реагування на змінні навантаження.
Підхід до використання таких інструментів, як Amazon EC2, S3, RDS, DynamoDB, CloudFront, Lambda, а також комплексна система безпеки через IAM, WAF і Shield, забезпечує AWS стабільну та відмовостійку архітектуру. Інтеграція з CloudFormation, CodePipeline та іншими DevOps-рішеннями дозволяє автоматизувати процеси розгортання та обслуговування застосунків, підвищуючи ефективність управління інфраструктурою.
Загалом, архітектура AWS демонструє правильне та сучасне застосування хмарних технологій для побудови масштабованої, продуктивної та безпечної екосистеми, здатної обробляти величезні обсяги даних, підтримувати різноманітні ІТ-сценарії та забезпечувати безперервну роботу критично важливих бізнес-процесів у глобальному масштабі.
